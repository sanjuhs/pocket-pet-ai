# Roadmap

## Now — Stage 1 Foundation

- Publish a polished, accessible landing page and versioned white paper.
- Establish `uv`-managed research code, tests, and an educational notebook.
- Select license-compatible small model candidates and pin revisions.
- Build storage, KV-cache, and ternary-kernel reference experiments.
- Define benchmark metadata and machine-readable result schemas.
- Demonstrate structured action proposal and deterministic confirmation policy.

## Next — Stage 1 Evidence

- Run repeatable commodity-hardware baselines across a small candidate set.
- Add quality proxies alongside speed and memory measurements.
- Profile attention, feed-forward, memory traffic, and decode bottlenecks.
- Validate low-bit conversions and document failure modes.
- Refine the white paper using measured results, replacing estimates where possible.

## Then — Stage 2 FPGA Feasibility

- Freeze a bit-accurate kernel contract and tensor shape.
- Implement, simulate, synthesize, and—when hardware is available—measure a ternary compute kernel.
- Explore one memory-path optimization at a time.
- Publish reproducible utilization, timing, power-method, accuracy, and transfer-overhead results.
- Decide whether evidence supports scaling, architectural revision, or stopping.

## Later — Product Research

- Permission-scoped Android and desktop companions.
- Local speech and event-driven vision pipelines.
- Encrypted long-term personal memory and user controls.
- Integrated FPGA dataflow followed, conditionally, by chiplet/ASIC feasibility.
- Thermal, battery, radio, industrial-design, privacy, accessibility, and regulatory work.

No roadmap item implies completion or a shipment commitment. Advancement depends on the acceptance criteria in `docs/stages.md`.
